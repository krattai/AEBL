Het hulpmiddel van de bevestiging van dit formulier kunt controleren vereiste, geheel getal, e-mail en datum velden in uw formulieren. Het is gemakkelijk draagbaar aan uw formulieren en zal helpen redden serverzijde middelen uit vorm veld validatie en pagina opnieuw geladen. Alle berichten worden aangepast in het verborgen formulier element waar u ook de eis van validatie selecteren. U kunt mengen het vereiste element met een der andere elementen overtreft. Deze code is volledig draagbaar en kan worden gebruikt met elke vorm door gewoon uw verborgen elementen op de juiste manier toe te voegen en toe te voegen de onClick opdracht aan een knop.

Volgens de regels.
1) Instellen van een vereist veld gebruik uw bestaande veldnaam en toevoegen een r_ naar het begin van het veld naam en creÃƒÂ«ren van een verborgen element.
IN HET VOLGENDE VOORBEELD:
<input type="text" name="your_fieldname">
<input type="hidden" name="r_your_fieldname" value="Required error message goes here">

2) Om in te stellen integer veld gebruik van uw bestaande veldnaam en toevoegen een i_ naar het begin van het veld naam en creÃƒÂ«ren van een verborgen element.
IN HET VOLGENDE VOORBEELD:
<input type="text" name="your_fieldname">
<input type="hidden" name="i_your_fieldname" value="Integer error message goes here">

3) To set a date field use your existing field name and add an d_ to the beginning of the field name and create a hidden element. This date field only accepts the format MM/DD/YY or M/D/YY.
IN HET VOLGENDE VOORBEELD:
<input type="text" name="your_fieldname">
<input type="hidden" name="d_your_fieldname" value="Date error message goes here">

4) Een e-mail veld gebruiken om te valideren uw bestaande veldnaam en een e_ aan het begin van de naam van het veld toevoegen en maak een verborgen element.
IN HET VOLGENDE VOORBEELD:
<input type="text" name="your_fieldname">
<input type="hidden" name="e_your_fieldname" value="The email address format is incorrect">

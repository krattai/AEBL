This form validation tool can check for required, integer, email and date fields in your forms. It is easily portable to all your forms and will help save server side resources from form field validation and page reloads. All the messages are customized in the hidden form element where you also select the validation requirement. You can mix the required element with any of the other elements. This code is completely portable and can be used with any form by just adding your hidden elements appropriately and adding the onClick command to a button.

Following the rules.
1) To set a required field use your existing field name and add an r_ to the beginning of the field name and create a hidden element.
EXAMPLE:
<input type="text" name="your_fieldname">
<input type="hidden" name="r_your_fieldname" value="Required error message goes here">

2) To set an integer field use your existing field name and add an i_ to the beginning of the field name and create a hidden element.
EXAMPLE:
<input type="text" name="your_fieldname">
<input type="hidden" name="i_your_fieldname" value="Integer error message goes here">

3) To set a date field use your existing field name and add an d_ to the beginning of the field name and create a hidden element. This date field only accepts the format MM/DD/YY or M/D/YY.
EXAMPLE:
<input type="text" name="your_fieldname">
<input type="hidden" name="d_your_fieldname" value="Date error message goes here">

4) To validate an email field use your existing field name and add an e_ to the beginning of the field name and create a hidden element.
EXAMPLE:
<input type="text" name="your_fieldname">
<input type="hidden" name="e_your_fieldname" value="The email address format is incorrect">

Initialization of the script.
You MUST use the type="button" NOT THE type="submit" for your SUBMIT button, the javascript sends the submit for you.
EXAMPLE:
<input type=button Value="Button Message Here" onclick="check(form,form.elements.length)">
